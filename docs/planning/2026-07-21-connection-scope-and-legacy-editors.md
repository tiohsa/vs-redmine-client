# 接続スコープ固定と旧形式編集ファイル対応

日付: 2026-07-21
ステータス: 実装済み
プロジェクトルート: `/home/glorydays/projects/src/ts/pr`

## 依頼内容

新規チケット・新規コメント作成の開始時接続スコープを非同期処理全体で固定し、接続切替時の誤送信を防ぐ。旧形式編集ファイルは現在接続先へ移行し、回帰テスト、CI、バージョン、PR状態を整える。

## 背景

`syncToRedmine.ts` は開始時の `operationScope` を取得しているが、新規作成後のエディター書換え・登録経路と旧形式ファイルの復元判定に追加確認が必要である。`ticketEditorRegistry.ts` は接続ハッシュを持たない編集ファイルを `unresolved:unknown` として登録している。

## 調査結果

- 新規チケットは `syncNewTicketDraft()` から `rewriteNewTicketEditorToTicketMode()` を呼び、登録時のスコープ伝播を確認する必要がある。
- 新規コメントは `syncNewCommentDraft()` の `onCreated` から `finalizeNewCommentDraftDocument()` を呼ぶ。
- `resolveEditorConnectionScope()` は旧形式ファイルに `unresolved:unknown` を返す。
- CodeGraph は未初期化だったため、既存ソースとテストを根拠に実装する。
- 依存関係未導入のため、初回 `pnpm test` は TypeScript の `vscode` 型解決エラーで失敗した。

## 判断

- 旧形式ファイルは現在接続先へ移行する。明示割当UIの追加は今回の範囲外とする。
- 接続切替後も開始時スコープで状態を更新し、現在接続との不一致は既存の同期ガードに委ねる。

## 実装計画

1. 旧形式スコープ解決を現在スコープ移行へ変更する。
2. 新規作成処理のスコープ伝播を確認・修正し、接続切替回帰テストを追加する。
3. バージョン更新とCI検証を行い、計画結果を追記する。

## リスクと対策

- リスク: 接続切替後に既存接続のエディターが現在接続へ誤登録される。対策: 開始スコープを明示引数で伝播し、テストで登録値を検証する。
- リスク: 依存関係未導入でCI検証できない。対策: 依存関係を導入してから全検証を再実行する。

## 検証方法

- `pnpm test`
- `pnpm run lint`
- `pnpm run compile`
- 旧形式および異なる接続ハッシュの登録テスト

## 結果

新規チケット作成（Markdownヘッダー経路を含む）と新規コメント作成の開始スコープ固定、旧形式ファイルの現在接続先移行、回帰テスト、`0.5.4` 更新を実装した。`pnpm test` は757件すべて成功し、lint・compileも成功した。PR本文更新、Draft解除、GitHub上の最終レビューはGitHub API接続障害のため未実施。
